01 · Cover Letter

Describe an AI workflow you built that solved a real problem.

"What platforms did you orchestrate? What was the stuck point that prompted you to build it? And how did you package it — or how would you, if asked to share it with others?"

The most recent thing I built end-to-end is CampusVoice — an agent that lets students, counselors, and admins ask plain-English questions like "What are UTK students complaining about?" and get back a synthesized answer with real, verbatim quotes, instead of scrolling through thousands of Rate My Professors and College Confidential posts.

The stuck point: I had ~13K reviews indexed in Elasticsearch, but plain keyword search just dumps a wall of results — nobody wants to read 200 hits. I needed something that could decide what to search, retry when a query came back empty, and then write a coherent, quote-grounded answer.

What I orchestrated: Gemini 2.5 Flash as the reasoning loop, the Elastic MCP Server as the tool layer (Gemini calls Elasticsearch directly via JSON-RPC — no hardcoded query templates), and Elasticsearch Serverless on Elastic Cloud as the index. The official MCP SDK kept hanging on stdio on Windows, so I wrote a minimal JSON-RPC client myself — that was the actual unblock.

Packaging: a Flask API + vanilla JS frontend, Dockerized and deployed on Cloud Run, with a 3-tier search fallback baked into the system prompt so the agent never just gives up. If I were handing this to someone else, I'd share the GitHub repo and the architecture PDF — the system prompt itself is basically the workflow's blueprint (see the downloadable artifact in the AI Builds section below).

04 · Last New AI Feature / Platform Absorbed

Next.js, Supabase & Railway

The last real stack jump was building AIHackTracker for the Mind the Product hackathon (World Product Day 2026, run by Novus.ai). Everything I'd shipped before — including CampusVoice — was Python: Flask, Elasticsearch, Docker, Cloud Run. AIHackTracker meant React components, file-based routing in Next.js, a Postgres database via Supabase, and Gemini 1.5 Flash summarizing hackathon listings pulled from the Devpost API.

It took about three or four focused days before it stopped feeling foreign — the Jun 15 deadline forced the pace. What got me there fastest was Railway's push-to-deploy: every commit went live within a minute, so I could see exactly what a Next.js page or Supabase query actually did instead of reading about it in the abstract. By day two I was comfortable enough to start layering in the "what wins" intelligence that's the actual point of the tool.
